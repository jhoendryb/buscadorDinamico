Errores

El componente incluye un sistema centralizado de gestión de errores con códigos específicos y soluciones documentadas.

Tipos de error

El componente lanza SearchError con códigos específicos:

class SearchError extends Error {
    code: ErrorCode;
    solution: string;
    context?: Record<string, any>;
}

Códigos de error

Código Descripción Solución
SEARCH_001 El parámetro element es requerido Proporciona el selector CSS del contenedor donde se renderizará el buscador. Ejemplo: { element: '.mi-buscador' }
SEARCH_002 El parámetro element debe ser un string con el selector CSS El parámetro element debe ser un string válido de selector CSS. Ejemplo: '.mi-buscador', '#buscador-id', '[data-search]'
SEARCH_003 El parámetro fetch.url es requerido cuando procesServer es true Configura la URL del endpoint de búsqueda. Ejemplo: { fetch: { url: '/api/search' } }
SEARCH_004 El parámetro itemsPerPage debe ser un número El parámetro itemsPerPage debe ser de tipo number. Ejemplo: { itemsPerPage: 10 }
SEARCH_005 El parámetro itemsPerPage debe ser mayor a 0 El parámetro itemsPerPage debe ser un número positivo mayor a 0. Ejemplo: { itemsPerPage: 10 }
SEARCH_006 El parámetro tiene un formato de tipo inválido Verifica que el parámetro tenga el formato correcto.
SEARCH_010 No existe el contenedor especificado Verifica que el selector CSS sea correcto y que el elemento exista en el DOM antes de inicializar el componente.
SEARCH_011 No existe el contenedor principal Asegúrate de que el elemento especificado exista en el DOM antes de inicializar el componente.
SEARCH_020 Error de conexión al servidor Verifica tu conexión a internet y que el servidor esté disponible.
SEARCH_021 Error al obtener datos del servidor Verifica que la URL del endpoint sea correcta y que el servidor responda con el formato esperado.
SEARCH_030 Formato de datos inválido Verifica que los datos tengan el formato esperado por el componente.
SEARCH_031 La respuesta del servidor está vacía Verifica que el endpoint devuelva datos en el formato esperado.
SEARCH_040 Error al inicializar el componente Verifica la configuración y los parámetros proporcionados.
SEARCH_041 Error al renderizar el componente Verifica que el DOM esté disponible y que no haya conflictos con otros componentes.

Manejo de errores

Try/Catch

try {
    const search = new BuscadorDinamico.Search({
        element: '#app',
        data: items
    }).init();
} catch (error) {
    if (error.code) {
        console.error(`Error ${error.code}: ${error.message}`);
        console.error('Solución:', error.solution);
    } else {
        console.error('Error inesperado:', error);
    }
}

Evento de error

search.on('error', (error) => {
    console.error('Error capturado:', error);
    // Mostrar notificación al usuario
    showToast(error.message);
});

Modo desarrollo

Activa el modo desarrollo para logs detallados en consola:

const search = new BuscadorDinamico.Search({
    element: '#app',
    data: items,
    developmentMode: true // Activa logs detallados
}).init();

En modo desarrollo, el componente imprime:

  • Validación de parámetros
  • Timing de búsquedas
  • Estadísticas de caché
  • Detalles de peticiones HTTP

Errores de red (modo servidor)

// El componente maneja automáticamente:
// - Timeouts (configurable via fetch.timeout)
// - Errores HTTP (4xx, 5xx)
// - Errores de parseo JSON
// - AbortController para cancelar peticiones

const search = new BuscadorDinamico.Search({
    element: '#app',
    procesServer: true,
    fetch: {
        url: 'https://api.ejemplo.com/buscar',
        timeout: 10000 // 10 segundos
    }
}).init();
🚨
Errores fatales

Ciertos errores como ELEMENT_REQUIRED y ELEMENT_NOT_FOUND son fatales y impiden la inicialización. Asegúrate de validar el DOM antes de crear la instancia.
