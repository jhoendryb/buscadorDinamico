Configuración

El componente se configura mediante un objeto de parámetros pasado al constructor.

Parámetros básicos

const search = new BuscadorDinamico.Search({
    element: '#mi-buscador',     // Required: selector del contenedor
    data: [],                     // Datos iniciales (modo local)
    theme: 'adaptative',          // Tema CSS
    itemsPerPage: 10,             // Items por página
    debounceTime: 500,            // Delay de búsqueda en ms
    keyboardEnabled: false,       // Navegación por teclado
    cacheEnabled: false,          // Habilitar caché LRU
    zIndex: 1000,                 // Z-index del componente
    dom: 'scip', // Orden: s=search, c=content, i=items, p=pagination
    developmentMode: false,       // Modo desarrollo (logs detallados)
    highlightEnabled: false,      // Resaltar términos de búsqueda
    highlightClass: ''            // Clase CSS adicional para resaltado
}).init();

Referencia de parámetros

Parámetro Tipo Default Descripción
element string — Requerido. Selector CSS del contenedor
data Array [] Array de objetos con los datos a buscar
theme string 'adaptative' Tema CSS: adaptative, clean-white, blue-black, onyx-black, forest-green
itemsPerPage number 10 Cantidad de items por página
debounceTime number 500 Tiempo de espera en ms antes de ejecutar la búsqueda
keyboardEnabled boolean false Habilitar navegación con teclado (flechas, Enter)
cacheEnabled boolean false Habilitar caché LRU de resultados
cacheMaxSize number 50 Número máximo de entradas en caché
cacheTtlSeconds number 300 Tiempo de vida de la caché en segundos
zIndex number 1000 Z-index del componente
dom string 'scip' Orden de renderizado: s=search, c=content, i=items, p=pagination. Usa DomComponent enum en TypeScript
developmentMode boolean false Activa logs detallados y validaciones extra
highlightEnabled boolean false Habilita el resaltado del término de búsqueda en los resultados
highlightClass string '' Clase CSS adicional para el elemento de resaltado
responseAdapter Function null Función para transformar la respuesta del servidor al formato { data, countPage }
sortBy string null Campo por el cual ordenar inicialmente
sortOrder 'asc' | 'desc' 'asc' Dirección del ordenamiento
template string | Function null Plantilla personalizada para renderizar items
translation Object {} Traducciones personalizadas (ver sección i18n)

Configuración del servidor (fetch)

Para búsquedas remotas, usa el parámetro fetch:

const search = new BuscadorDinamico.Search({
    element: '#mi-buscador',
    procesServer: true,
    fetch: {
        url: 'https://api.ejemplo.com/buscar',
        method: 'POST',
        headers: {
            'Content-Type': 'application/json'
        },
        body: {
            query: '',
            page: 1,
            limit: 10
        },
        timeout: 10000
    }
}).init();

Múltiples instancias

Puedes crear múltiples buscadores independientes en la misma página:

const search1 = new BuscadorDinamico.Search({
    element: '#buscador-1',
    data: productos,
    theme: 'clean-white'
}).init();

const search2 = new BuscadorDinamico.Search({
    element: '#buscador-2',
    data: usuarios,
    theme: 'onyx-black',
    itemsPerPage: 5
}).init();
⚠️
Validación de parámetros

El componente valida los parámetros en el constructor. Si element no es un string o itemsPerPage es menor a 1, se lanzará un SearchError.
