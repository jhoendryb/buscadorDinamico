Changelog

Historial de versiones y cambios del componente.

v1.0.0 Latest

Fecha: 2026

Añadido

  • SearchEventMap — Mapa tipado de eventos que asocia cada nombre con su tipo de datos
  • EventEmitter<T> genérico con type-safety completo (EventEmitter<SearchEventMap>)
  • DomComponent enum para controlar el orden de renderizado (SEARCH, CONTENT, ITEMS, PAGINATION)
  • SearchResult interface { data: Record<string, any>[]; countPage?: number }
  • PaginationRange interface { from: number; to: number; total: number }
  • Método Pagination.getRange() para el rango actual de items
  • Método Pagination.getTotalPages() para total de páginas
  • Método Pagination.getTotalLoaded() para items cargados
  • Método Pagination.loadNextPage() para avanzar página
  • Método Pagination.hasMorePages() para verificar páginas restantes
  • Método EventEmitter.emitAsync() para eventos asíncronos
  • Método EventEmitter.getEventList() para lista detallada de eventos
  • Método SearchRenderer.destroy() para limpiar timeouts y referencias DOM
  • Método SearchRenderer.toggleResults() para toggle de visibilidad
  • Método SearchRenderer.hideResultsWithDelay(delay?)
  • Parámetro loadingText en showLoading(text) para texto de carga personalizado
  • Propiedad LRUCache.stats — estadísticas de caché (hits, misses, evictions)
  • Método LRUCache.getOrFetch(key, fetch, onMiss?)
  • Método LRUCache.delete(key) y LRUCache.size()
  • Clave pagination en traducciones para personalizar el contador
  • ItemSelectedEventData.close — método para cerrar resultados
  • Interfaces completas: SearchEventData, ErrorData, ResultsClearedEventData, AppendItemsEventData

Mejorado

  • Search.on() tipado con genéricos: on<K extends keyof SearchEventMap>(...)
  • SearchRenderer.updateCounter() acepta objeto { from, to, total, textPagination? }
  • EventEmitter constructor acepta ErrorHandler opcional
  • Manejo de errores en callbacks: errores individuales no afectan a otros listeners
  • Traducción por defecto pagination: "{{to}} de {{total}}"

v2.3.0

Fecha: 2026

Añadido

  • Parámetro responseAdapter para transformar respuestas del servidor personalizadas
  • Método clear() para resetear estado sin destruir la instancia
  • Evento resultsCleared emitido al cambiar término de búsqueda
  • Evento searchComplete emitido al completar búsqueda y renderizado
  • Búsqueda recursiva en objetos anidados (flattenValues)
  • Normalización Unicode para búsqueda con tildes/diacriticos
  • Protección contra condiciones de carrera en draw()
  • Protección contra re-entrancy en #loadMore()
  • Estilos de scrollbar cross-browser para Firefox
  • prefers-reduced-motion ahora desactiva TODAS las transiciones globalmente

Mejorado

  • draw() ahora retorna Promise<Search>
  • cache.has() refresca TTL (Time To Live) al acceder (true LRU)
  • setItemsPerPage() valida que el valor sea >= 1
  • hideResults() usa timeHiddenResults configurable
  • destroy() re-inicializa objetos en lugar de nullificarlos
  • destroy() remueve event listeners por referencia
  • #highlightText() escapa caracteres especiales de regex
  • Typo corregido: firtsLoad > firstLoad

Corregido

  • highlightText() escapa caracteres regex especiales
  • Cache almacena _data completa en server mode
  • clearCacheByPrefix() valida searchTerm vacío
  • processInfiniteScroll() ya no se llama desde searchingServer

v2.2.0

Fecha: 2026

Características iniciales

  • Búsqueda local en tiempo real con debounce configurable
  • Búsqueda remota vía Fetch API con soporte para JSON, FormData y URL-encoded
  • Paginación por scroll infinito con Intersection Observer
  • Sistema de caché LRU (Least Recently Used) con TTL (Time To Live) configurable
  • 5 temas CSS predefinidos (adaptative, clean-white, blue-black, onyx-black, forest-green)
  • Sistema de eventos personalizado con 8 eventos
  • Navegación por teclado (flechas arriba/abajo, Enter)
  • Plantillas personalizables (string y funciones)
  • Sistema i18n con traducciones configurables
  • Gestión centralizada de errores con 14 códigos específicos
  • Soporte completo de TypeScript con tipos exportados
  • Arquitectura modular OOP con 8 clases independientes
  • Build dual: ES Module + UMD
  • Accesibilidad: roles ARIA, navegación por teclado, labels
  • Soporte para múltiples instancias en la misma página

Temas CSS

  • adaptative — Se adapta al tema del sistema (light/dark)
  • clean-white — Diseño limpio y minimalista en blanco
  • blue-black — Tema oscuro con acentos azules
  • onyx-black — Tema completamente oscuro
  • forest-green — Tema oscuro con tonos verdes

Eventos

  • init — Emite al inicializar el componente
  • search — Emite después de cada búsqueda
  • pageChange — Emite al cambiar de página
  • sortChange — Emite al cambiar el ordenamiento
  • itemSelected — Emite al seleccionar un item
  • itemHighlighted — Emite al destacar un item
  • appendItems — Emite al añadir items al DOM
  • destroy — Emite antes de destruir la instancia

Códigos de error

  • SEARCH_001 — element requerido
  • SEARCH_002 — element debe ser string
  • SEARCH_003 — fetch.url requerido en modo servidor
  • SEARCH_004 — itemsPerPage debe ser número
  • SEARCH_005 — itemsPerPage debe ser mayor a 0
  • SEARCH_006 — tipo de parámetro inválido
  • SEARCH_010 — elemento contenedor no encontrado
  • SEARCH_011 — contenedor principal no encontrado
  • SEARCH_020 — error de conexión al servidor
  • SEARCH_021 — error al obtener datos del servidor
  • SEARCH_030 — formato de datos inválido
  • SEARCH_031 — respuesta del servidor vacía
  • SEARCH_040 — error al inicializar el componente
  • SEARCH_041 — error al renderizar el componente
