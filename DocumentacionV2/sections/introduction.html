Introducción

Buscador Dinámico es un componente de búsqueda modular, ligero y altamente personalizable escrito en TypeScript. Está diseñado para integrarse fácilmente en cualquier proyecto web, ofreciendo búsqueda local o remota con soporte para scroll infinito, caché LRU y internacionalización.

💡
Características principales

Búsqueda en tiempo real, paginación por scroll infinito, sistema de eventos, múltiples temas CSS y soporte completo de TypeScript.

¿Por qué Buscador Dinámico?

Las soluciones de búsqueda existentes suelen ser demasiado genéricas o dependientes de frameworks pesados. Buscador Dinámico ofrece un enfoque diferente:

  • Sin dependencias — Vanilla JavaScript/TypeScript, funciona con cualquier framework o sin ellos.
  • Modular — Arquitectura OOP con clases independientes que se pueden extender o reemplazar.
  • Performante — Caché LRU, debounce configurable, scroll infinito con Intersection Observer.
  • Accesible — Navegación por teclado, roles ARIA, soporte para lectores de pantalla.
  • Personalizable — Templates, temas CSS con variables, sistema i18n integrado.
  • Resaltado de texto — Opcional, resalta los términos de búsqueda en los resultados.
  • Búsqueda profunda — Busca recursivamente en objetos anidados con normalización Unicode.
  • Adapter de respuesta — Transforma respuestas del servidor con responseAdapter.

Arquitectura

El componente sigue un patrón de composición OOP con 8 módulos principales:

Módulo Responsabilidad
Search Clase principal orquestadora
SearchRenderer Renderizado del DOM y gestión visual
SearchingLocal Búsqueda en memoria local
SearchingServer Búsqueda remota vía Fetch API
Pagination Gestión de paginación y scroll infinito
LRUCache Caché de resultados con TTL
EventEmitter Sistema de eventos personalizado
ErrorHandler Gestión centralizada de errores

Modos de uso

El componente soporta múltiples modos de operación:

  1. Búsqueda local — Datos en memoria, ideal para listas pequeñas/medianas.
  2. Búsqueda desde DOM — Extrae datos automáticamente de elementos HTML existentes.
  3. Búsqueda remota — Peticiones HTTP con Fetch API, soporte para JSON, FormData y URL-encoded.
  4. Instancias múltiples — Varios buscadores independientes en la misma página.

Requisitos mínimos

  • Navegador moderno (Chrome 80+, Firefox 78+, Safari 14+, Edge 80+)
  • Soporte para ES2020 (para uso directo) o cualquier bundler moderno
  • Intersection Observer API (para scroll infinito)
