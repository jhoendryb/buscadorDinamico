Caché

El sistema de caché LRU (Least Recently Used) almacena resultados de búsquedas anteriores para mejorar el rendimiento.

Habilitar caché

const search = new BuscadorDinamico.Search({
    element: '#app',
    data: items,
    cacheEnabled: true,
    cacheMaxSize: 50,        // Máximo 50 entradas
    cacheTtlSeconds: 300     // Expira después de 5 minutos
}).init();

Configuración

Parámetro Tipo Default Descripción
cacheEnabled boolean false Habilita el sistema de caché
cacheMaxSize number 50 Número máximo de entradas en caché
cacheTtlSeconds number 300 Tiempo de vida de cada entrada en segundos

Cómo funciona

  1. Al ejecutar una búsqueda, se genera una clave basada en el término y la página
  2. Si la clave existe en caché y no ha expirado, se retorna el resultado almacenado
  3. Si no existe o expiró, se ejecuta la búsqueda normal y se almacena el resultado
  4. Al superar el tamaño máximo, se eliminan las entradas menos usadas

Generación de claves

// La clave se genera automáticamente como: "término_página"
// Ejemplo: "laptop_1", "laptop_2", "tablet_1"

const key = search.getCacheKey('laptop', 1);
console.log(key); // "laptop_1"

Limpiar caché

// Limpiar toda la caché
search.cache.clear();

// Limpiar entradas con un prefijo específico
search.cache.clearCacheByPrefix('laptop');

// Eliminar una entrada específica
search.cache.delete('laptop_1');

// Verificar tamaño actual
console.log('Entradas en caché:', search.cache.size());

Uso con datos del servidor

La caché es especialmente útil con búsquedas remotas para reducir peticiones HTTP:

const search = new BuscadorDinamico.Search({
    element: '#app',
    procesServer: true,
    cacheEnabled: true,
    cacheMaxSize: 100,
    cacheTtlSeconds: 600, // 10 minutos
    fetch: {
        url: 'https://api.ejemplo.com/buscar',
        method: 'POST'
    }
}).init();

// Las búsquedas repetidas usarán caché
// "laptop" -> petición HTTP
// "laptop" -> resultado de caché (sin petición)
💡
Rendimiento

La caché LRU es ideal para patrones de búsqueda donde los usuarios tienden a repetir términos. Reduce significativamente la carga en servidores y mejora la experiencia de usuario.
